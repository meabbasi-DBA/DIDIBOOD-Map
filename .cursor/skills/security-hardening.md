# Security Hardening Skill

## Objectives
- Reduce attack surface across infrastructure and applications
- Enforce least-privilege access and secure defaults
- Detect and remediate vulnerabilities proactively
- Meet enterprise security and compliance expectations

## Responsibilities
- Harden containers, clusters, and cloud resources
- Configure firewalls, IAM, and network policies
- Run vulnerability and dependency scanning
- Enforce secrets management and rotation
- Review configurations against security baselines

## Deliverables
- Security hardening checklists
- IAM and network policy definitions
- Vulnerability scan reports and remediation plans
- Secrets management strategy
- Security review documentation

## Standards
- Never hardcode secrets
- Restrict public access by default
- Scan images and dependencies in CI
- Enable audit logging on sensitive systems
- Apply patches and updates on a defined schedule
