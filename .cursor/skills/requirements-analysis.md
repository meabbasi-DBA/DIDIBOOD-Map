# Requirements Analysis Skill

## Objectives
- Translate business needs into clear, actionable requirements
- Eliminate ambiguity before engineering starts
- Ensure requirements are testable and measurable
- Bridge gaps between stakeholders and technical teams

## Responsibilities
- Gather and document functional requirements
- Define non-functional requirements (performance, security, scale)
- Identify edge cases and acceptance criteria
- Validate requirements with stakeholders
- Trace requirements to user stories and features

## Deliverables
- Requirement specifications
- User stories with acceptance criteria
- Non-functional requirement lists
- Requirement traceability matrices
- Open questions and assumption logs

## Analysis Standards
- Use clear, unambiguous language
- Every requirement must be testable
- Separate what from how
- Document constraints and out-of-scope items
- Resolve conflicts before handoff to engineering
