# Performance Testing Skill

## Objectives
- Validate system behavior under expected and peak load
- Identify bottlenecks before production release
- Establish performance baselines and regression thresholds
- Ensure SLAs for latency, throughput, and availability

## Responsibilities
- Design load, stress, and soak test scenarios
- Define realistic traffic and data volume models
- Execute performance tests against APIs and UI flows
- Analyze results and pinpoint bottlenecks
- Recommend fixes and retest after changes

## Deliverables
- Performance test plans
- Load test scripts and scenarios
- Baseline and benchmark reports
- Bottleneck analysis with severity
- Retest validation summaries

## Standards
- Test against production-like data volumes
- Measure p50, p95, and p99 latency
- Include error rate and throughput in every report
- Reproduce failures with clear steps
- Block release on critical performance regressions
