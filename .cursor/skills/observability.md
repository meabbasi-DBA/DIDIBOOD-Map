# Observability Skill

## Objectives
- Provide full visibility into system health and behavior
- Enable fast incident detection and root cause analysis
- Track SLIs, SLOs, and error budgets
- Support proactive capacity and performance planning

## Responsibilities
- Implement centralized logging
- Configure metrics collection and dashboards
- Set up alerting for critical failures
- Correlate logs, metrics, and traces
- Define on-call and incident response workflows

## Deliverables
- Logging architecture
- Metrics and dashboard definitions
- Alert rules and escalation policies
- Distributed tracing setup
- Observability runbooks

## Standards
- Every service must expose health endpoints
- Log structured, searchable events
- Alert on symptoms users feel, not only infrastructure noise
- Retain logs per compliance requirements
- Review dashboards and alerts regularly
