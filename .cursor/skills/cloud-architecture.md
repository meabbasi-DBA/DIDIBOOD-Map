# Cloud Architecture Skill

## Objectives
- Design scalable, secure cloud-native platforms
- Select appropriate managed services for each workload
- Optimize cost, performance, and reliability trade-offs
- Plan for multi-environment and disaster recovery needs

## Responsibilities
- Define cloud reference architectures
- Design network topology and access boundaries
- Plan storage, compute, and data service layouts
- Evaluate HA, DR, and failover strategies
- Align architecture with security and compliance requirements

## Deliverables
- Architecture diagrams
- Service selection rationale
- Network and security zone designs
- HA and DR plans
- Capacity and cost estimates

## Standards
- Prefer managed services where they reduce operational burden
- Design for horizontal scalability
- Isolate environments and restrict public exposure
- Document architectural decisions and trade-offs
- Plan disaster recovery before production launch
